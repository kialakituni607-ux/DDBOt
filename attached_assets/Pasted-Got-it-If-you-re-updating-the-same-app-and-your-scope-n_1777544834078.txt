Here’s a simple authorize flow for the token path:

1) Collect token
- Let user choose “Use API token” and paste their personal token
- Validate it’s non-empty; don’t store it on disk

2) Open API connection
- Connect to Deriv API (see docs below)

3) Authorize with token
- Send an authorize request using the provided token
- Wait for a successful auth response (session is now active)

4) Post-auth checks
- Optionally fetch basic account info to confirm login
- Proceed with your normal flow (proposal, buy, ticks, etc.)

5) Error handling
- Invalid token: prompt user to re-enter
- Missing scope: tell user which permission is missing and ask them to create a new token with only those scopes
- Revoked/expired: prompt for a fresh token
- Auth failure: offer to switch back to OAuth as fallback

6) Session & logout
- Keep token only in memory/session
- On user sign-out or app close, send a logout request and clear session

7) Security tips
- Use least-privilege scopes for tokens
- Never log tokens; mask in any diagnostics
- Offer an in-app “Replace token” flow and handle 401 by re-auth
- Test the token path and your OAuth path separately, then together
